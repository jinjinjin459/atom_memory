# rag_demo.py README

## 역할 (Role)
API 서버를 띄우지 않고 로컬 콘솔(터미널)에서 전체 하이브리드 RAG 사이클을 검증 및 벤치마킹하는 **메인 테스트 스크립트 (CLI Demo)** 입니다.

## 수행 시나리오
1. 모의 데이터 5종(인사부 연차 관련, IT부서 VPN 관련, 예산, 시험 관련 문서 등)을 메모리 기반으로 인덱싱합니다.
2. **테스트 1 (RAG 정상 동작)** : 연차 수당에 관한 질의 -> 검색 & LLM 생성 응답.
3. **테스트 2 (캐시 히트 확인)** : 테스트 1과 동일한 질문 입력 -> LLM을 타지 않고 즉시 0ms 지연으로 응답(`cache_hit`).
4. **테스트 3~4 (일반 질의)** : 예산 변동 확인 및 시험 출제 범위에 대한 개별 질문 처리 테스트.

## 협업 가이드 (Collaboration Guide)
1. RAG 모듈 내부 로직을 수정하고 빌드/커밋하기 전에, 항상 `python rag_demo.py`를 실행하여 에러가 터지지 않는지 확인(Sanity Check) 용도로 사용하세요.
2. 특정 도메인 데이터 검색이 잘 되는지 확인해야 할 때, 이 스크립트의 `sample_documents` 변수에 테스트 문구들을 넣어보고 튜닝하면 편리합니다.
