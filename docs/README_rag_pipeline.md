# rag_pipeline.py README

## 역할 (Role)
단일 파이프라인 안에서 1~4단계(캐시 -> 최적화 -> 하이브리드 검색 -> 리랭킹 -> 생성) 모듈들을 엮어서 통제하는 **오케스트레이터 (Orchestrator)** 역할의 파일입니다.

## 동작 흐름 메커니즘
1. `query(user_query)` 호출.
2. **(1단계)** `SemanticCache`를 조회합니다. 적중하면 2~4단계를 건너뛰고 바로 리턴.
3. **(2단계)** 캐시 미스 시 `QueryOptimizer`에 질의하여 검색 키워드를 얻습니다.
4. **(3단계)** `QdrantHybridSearch`로 찾아온 초기 문서를 가져옵니다.
5. **(4단계)** `Reranker`를 사용해 불필요한 문서를 날리고, `GeminiGenerator`로 최종 답변을 가져옵니다.
6. 완료된 답변은 추후 재사용을 위해 `SemanticCache`에 저장(Store)합니다.

## 협업 가이드 (Collaboration Guide)
1. 이 파일은 각 모듈들을 "조립"하는 파일입니다. 단일 모듈의 디테일한 버그 수정은 각 모듈 파일에서 진행하세요.
2. 중간 로깅 텍스트나 전체 흐름에서의 성능 측정(타임체크)을 개선하라면 이 파일의 `query()` 메서드를 수정하세요.
3. `query_stream()`은 실시간 스트리밍 답변을 위한 제너레이터 함수입니다. 응답 인터페이스 수정 시 이 부분도 유의하세요.
